The DevOps Lifecycle

Under a DevOps model, development and operations teams are no longer "siloed." Sometimes, these two teams are merged into a single team where the engineers work across the entire application lifecycle, from development and test to deployment to operations.

  • Speed: Move at high velocity to innovate for customers faster.
  • Reliability: Ensure the quality of application updates and infrastructure changes.
  • Scale: Operate and manage your infrastructure and development processes at scale.

Core DevOps Practices

The essential practices that enable organizations to innovate faster through automating and streamlining the software delivery process.

Continuous Integration

Developers regularly merge their code changes into a central repository, after which automated builds and tests are run to find and address bugs quicker.

Continuous Delivery

Code changes are automatically built, tested, and prepared for a release to production. It expands upon continuous integration by deploying all code changes.

Microservices

An architectural approach to building a single application as a set of small services. Each service runs in its own process and communicates via APIs.

Infrastructure as Code

Infrastructure is provisioned and managed using code and software development techniques, such as version control and continuous integration.

Monitoring & Logging

Organizations monitor metrics and logs to see how application and infrastructure performance impacts the experience of their product's end user.

Communication & Collab

Increased communication and collaboration is one of the key cultural aspects of DevOps. Tools like chat apps and issue tracking systems are crucial.
